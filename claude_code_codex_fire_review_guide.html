重要聲明

AI 審圖只能作為輔助檢查工具。最終判斷仍需由具備資格與責任的人員確認。AI 輸出的法規條文、距離計算、設備配置意見，均需要回到原圖、原始標註、正式法規與專業判斷進行複核。

本文件所稱「審圖」指的是「輔助初步檢查」，包含整理圖面、讀取標註、抽取設備座標、檢查常見缺漏、產生審查意見草稿。它不等同於正式審查、簽證或法定審核。

階段目的產出
前置準備建立固定資料夾與檔案格式可被 AI 穩定讀取的案件資料夾
Claude Code 路線以互動方式讀圖、問法規、產生審查意見適合單案、即時問答、人工主導
Codex 路線建立可重複執行的審圖工作流程適合批次案件、腳本分析、自動化輸出
人工複核檢查 AI 結論是否可靠最終審查意見修正版

Step 2：準備三種圖面格式

建議同一張圖面同時準備三種版本。每種格式的用途不同。

格式用途注意事項
PDF讓 AI 理解圖面整體配置、平面關係、標註內容輸出時需保持清晰，避免文字過小
DXF讓 AI 或輔助腳本抽取圖層、圖塊、座標、距離需確認單位、比例、圖層命名是否穩定
PNG 或 JPG讓 AI 檢查局部區域、設備配置、標註疑點建議針對局部放大截圖，不要只用全圖縮圖

DXF 適合做數值計算。PDF 與截圖適合做空間理解。審圖時兩者要搭配使用。

第二部分：方法一，使用 Claude Code 審圖

適合使用 Claude Code 的情境

Claude Code 適合由人主導、一邊看圖一邊問問題的工作方式。若你需要 AI 協助閱讀 PDF、截圖、圖面說明，並用對話方式逐步產生審查意見，Claude Code 是比較直覺的選擇。

適合情境：

  • 單一案件的初步審圖。
  • 需要用中文逐步詢問圖面疑點。
  • 需要 AI 同時閱讀 PDF、截圖、DXF 與文字說明。
  • 需要 AI 產生審查意見草稿，再由人工修改。
  • 不想先建立完整自動化腳本。

Step 12：人工複核 Codex 輸出

Codex 可能會建立腳本與中間檔。人工複核時需檢查：

  • 腳本是否讀取正確資料夾。
  • 有沒有誤刪或覆蓋原始檔。

第七部分：常見錯誤與處理方式

問題一：AI 把 DXF 當成看得見的圖

處理方式：提醒 AI，DXF 主要用於抽取圖層、圖塊、座標與文字。空間配置需搭配 PDF 或截圖判讀。

問題二：AI 直接下法規結論

處理方式：要求 AI 標示依據來源。無來源時列為「需確認」。

問題三：DXF 單位錯誤

處理方式：在案件說明中明確寫出單位。執行距離計算前，要求 AI 先確認單位。

問題四：設備圖塊辨識錯誤

處理方式：要求 AI 先列出圖塊名稱與判斷依據。人工確認後再進行數量與距離計算。

問題五：PDF 標註太小

處理方式：輸出局部放大截圖。每張截圖只包含一個區域或一個系統，避免全圖縮圖。

問題六：批次審圖結果混在一起

處理方式：每個案件使用獨立資料夾。輸出檔名必須包含案件代號。
