步驟 03

下載並安裝 Claude 桌面應用程式

從官方網頁下載 .exe，安裝方式與一般軟體完全相同 · 約 5 分鐘

Claude 桌面應用程式整合了聊天介面與 Claude Code 終端機，是新手最簡單的入門方式，不需要安裝 Node.js 或 Git，直接下載安裝即可。

無需任何前置安裝
桌面應用程式已內建所有必要環境。不需要 Node.js、不需要 Git、不需要 PowerShell 指令。

方式 A — 推薦新手首選：官方網頁下載頁面

步驟 3-1｜前往下載頁面
開啟瀏覽器，前往 Claude 官方下載頁面：https://claude.com/download

步驟 3-2｜點選 Windows 下載按鈕
在頁面找到「Desktop」區塊，點選 Windows 的「Download」按鈕。瀏覽器底部或右上角會顯示下載進度，檔名類似 Claude-Setup-x64.exe。

📺 畫面說明：頁面上方有「Download for Windows」大按鈕。一般電腦（Intel/AMD）選第一個；若電腦是 ARM 架構，選「Windows (arm64)」。不確定就選第一個。

步驟 3-3｜執行安裝
下載完成後，雙擊 Claude-Setup-x64.exe 執行。Windows 若出現「是否允許此應用程式變更您的裝置？」→ 點 「是」。安裝全自動，約 30–60 秒完成，不需要任何設定選項，結束後 Claude 會自動開啟。

安裝成功確認：桌面出現 Claude 圖示，或開始功能表搜尋「Claude」可找到，代表安裝完成。

方式 B — 備選（公司電腦限制 .exe 時）

若公司政策不允許執行 .exe 安裝檔，可改用官方 PowerShell 腳本。以系統管理員身份開啟 PowerShell，貼上以下指令按 Enter：

irm https://claude.ai/install.ps1 | iex

安裝完成後關閉並重新開啟終端機即可使用。

✓ 步驟 03 完成
桌面或開始功能表能看到 Claude 應用程式圖示即可繼續下一步。

步驟 05

建立消防審圖工作資料夾

用檔案總管建立，不需要任何指令 · 約 3 分鐘

建立固定的資料夾結構，讓 Claude Code 每次都能在同一個地方找到你的圖面和文件。

步驟 5-1：按 Windows 鍵 + E 開啟檔案總管，進入「本機 (C:)」。在空白處按右鍵 →「新增」→「資料夾」，命名為 消防審圖。

步驟 5-2：進入 C:\消防審圖，在裡面建立三個子資料夾：

  • 圖面輸入 — 放要審查的 DXF / PDF / 圖片
  • 審查報告輸出 — AI 產出的審查意見草稿
  • 參考資料 — 法規條文摘要等輔助文件

完成後的資料夾結構

C:\消防審圖\ ├── CLAUDE.md ← 下一步建立的 AI 設定檔 ├── 圖面輸入\ ← 放要審查的 DXF / PDF / 圖片 ├── 審查報告輸出\ ← AI 產出的審查意見草稿 └── 參考資料\ ← 法規條文摘要等輔助文件
📐 DXF、PDF、截圖各有用途，建議雙管齊下
AI 讀取 DXF 的方式是解析座標數字，無法「看見」圖面的空間樣貌。建議同時準備兩種格式：DXF 供 AI 提取座標做計算，PDF 或截圖 讓 AI 理解整體配置。詳細說明見步驟 08。
✓ 步驟 05 完成
C:\消防審圖 內有三個子資料夾即可繼續。
步驟 06

建立 CLAUDE.md 設定檔

告訴 AI 你的工作背景，之後不用每次重複說明 · 約 5 分鐘

CLAUDE.md 是什麼？

放在工作資料夾裡的文字檔，Claude Code 每次啟動都會自動讀取。把它想像成「給 AI 的工作說明書」——設定好後，AI 一開始就知道你是消防製圖人員、要審查哪些項目、用什麼格式輸出。

步驟 6-1｜用記事本建立檔案

按 Windows 鍵 搜尋「記事本」，開啟。先不要輸入任何內容，按 Ctrl + S 存檔：存檔位置選 C:\消防審圖，檔案名稱輸入 CLAUDE.md，存檔類型務必選 「所有檔案 (*.*)」（否則會存成 CLAUDE.md.txt），按儲存。

步驟 6-2｜貼入以下內容

將下方整個程式碼區塊的內容全選複製（Ctrl+A 全選，Ctrl+C 複製），貼到記事本中：

# 消防圖面輔助審查工作環境設定 ## 工作背景 - 我是消防設備製圖人員，負責消防安全設備圖面的繪製與初步審查 - 主要使用 AutoCAD，圖面格式包含 DXF、PDF 及圖片（JPG/PNG） - 適用法規：《各類場所消防安全設備設置標準》（台灣） ## 主要審查項目 - 自動撒水設備：灑水頭間距、保護面積、水平距離 - 火警自動警報設備：探測器間距、覆蓋半徑、類型選用 - 緊急廣播設備：揚聲器配置、音壓規定 - 消防栓：位置、有效水平距離 - 避難逃生：走廊寬度、出口數量、疏散路徑 ## 回答格式要求 - 請用繁體中文回答所有問題 - 發現疑義時，請同時引用設置標準的條文編號 - 審查結果請以「符合」「不符合」「需確認」三種狀態標示 - 數值計算需顯示計算過程，方便核對 - 輸出審查清單時，請使用表格格式 ## 資料夾說明 - 圖面輸入/：待審查的 AutoCAD 圖面及相關文件 - 審查報告輸出/：存放 AI 輔助生成的審查意見草稿 - 參考資料/：法規條文摘要、常見缺失範本等 ## 重要提醒 - AI 審查結果僅供輔助參考，最終判斷仍需具資格人員確認 - 如遇模糊地帶，請說明需要進一步核對的項目

步驟 6-3

按 Ctrl + S 存檔，確認 C:\消防審圖 中出現了 CLAUDE.md 檔案。

✓ 步驟 06 完成
C:\消防審圖\CLAUDE.md 存在即完成。之後 AI 每次啟動都會自動讀取這份設定。

步驟 08

消防審圖實戰範例

可直接複製使用的中文指令

在 claude> 提示符後直接輸入中文即可，不需要任何特殊語法。以下是針對消防製圖工作的常用場景。

DXF 檔案的能力說明與建議工作流程

DXF 可以做到
  • 讀取圖層名稱清單
  • 提取座標數值，計算設備間距
  • 統計圖元數量（LINE、CIRCLE、INSERT）
  • 讀取文字標註內容（TEXT / MTEXT）
  • 識別圖塊名稱（設備符號定義）
DXF 做不到
  • 視覺化「看見」圖面空間配置
  • 理解整體平面布局與動線
  • 辨識圖面美觀、比例感
  • 大型複雜 DXF 可能超出處理量

建議雙管齊下的工作方式

審查需求使用格式怎麼做
灑水頭間距、座標計算DXFAutoCAD 另存新檔 → DXF
圖層名稱、圖塊清單確認DXFAutoCAD 另存新檔 → DXF
整體平面配置審查PDFAutoCAD 列印 → 存成 PDF
局部配置說明、探測器位置截圖Windows + Shift + S 截圖存 PNG

實用建議：同一份圖同時準備 DXF 和 PDF 兩個版本，先用 PDF 讓 AI 理解整體配置，再用 DXF 做精確的數值計算，兩者互補效果最好。

範例一：列出圖面輸入資料夾中的所有檔案

claude> 請讀取「圖面輸入」資料夾中的所有檔案，列出檔案名稱並簡述圖面內容。
AI 回覆範例： 我已掃描「圖面輸入」資料夾，找到以下檔案： 1. 案件A_平面圖.dxf — AutoCAD DXF，B1 樓層平面與設備圖層 2. 案件A_灑水系統.pdf — PDF 圖說，含撒水管線配置 3. 案件B_探測器配置.png — 偵煙探測器位置分布圖 請問您想先從哪個檔案開始審查？

範例二：灑水頭間距驗算

claude> 圖面標示的灑水頭間距為 3.5m × 3.5m，閉式撒水頭，場所為辦公室。請驗算是否符合設置標準。
AI 回覆範例： 【灑水頭間距驗算結果】 ■ 適用條文：設置標準第 46 條 ■ 場所分類：一般保護場所（辦公室） ■ 法規最大間距：3.0m × 3.0m ■ 圖面標示：3.5m × 3.5m ■ 判斷：不符合 — 超過上限 3.0m，建議修改為 ≤ 3.0m

範例三：產生審查報告並存檔

claude> 請把剛才的審查內容整理成表格，存到「審查報告輸出」資料夾，檔名「案件A審查意見.md」。
AI 回覆範例： 已建立：審查報告輸出/案件A審查意見.md 審查項目共 8 項｜符合 5 項｜不符合 2 項｜需確認 1 項

更多常用指令

  • 法規查詢：「設置標準中，地下停車場的偵煙探測器安裝規定是哪一條？請摘要說明。」
  • 數量統計：「計算這份平面圖中的偵煙探測器總數量，確認與圖說標示數量一致。」
  • 圖面比對：「比對圖面A和圖面B，找出灑水系統配置的差異，用表格呈現。」
  • 缺漏清單：「這份圖說缺少哪些常見必要標示？請列出清單，存成 Excel。」

以 / 開頭的系統指令（在 claude> 提示符後輸入）：

遇到問題怎麼辦？

常見疑難排解

安裝後找不到 Claude 應用程式圖示

  • 按 Windows 鍵搜尋「Claude」，找到後可右鍵釘選到桌面
  • 找不到：重新啟動電腦後再搜尋
  • 仍無效：至 https://claude.com/download 重新下載安裝

開啟 App 後登入頁面白畫面或無法載入

  • 確認網路正常（試開 google.com）
  • 工作列右下角 Claude 圖示按右鍵 →「結束」，重新開啟
  • 公司電腦：確認 claude.ai 與 api.anthropic.com 未被防火牆封鎖

終端機輸入 claude 顯示「找不到命令」

  • 改用桌面 App 內建的 Claude Code 終端機，不需額外設定
  • 若要用外部 cmd：完全關閉視窗，重新開啟再試
  • 仍無效：重新啟動電腦讓 PATH 環境變數生效

顯示「Authentication failed」或「Unauthorized」

登入狀態已過期。輸入 /logout，再輸入 claude 重新登入。或在 App 設定中重新登入帳號。

顯示「Insufficient credits」或「Payment required」

訂閱有問題。登入 claude.ai，確認 Pro 訂閱狀態，或確認信用卡有正常扣款。

AI 回覆的中文出現亂碼

桌面 App 內建終端機通常不會有此問題。若用外部 cmd，在啟動前輸入 chcp 65001。建議改用 App 內建終端機，或安裝 Microsoft Store 免費的 Windows Terminal。

如何更新 Claude 到最新版本

桌面應用程式會自動在背景更新，通常不需要手動處理。若想立即更新，在 Claude App 選單中找「Check for updates」，或在終端機輸入 claude update。

還是無法解決？
前往 https://support.claude.com 官方說明，或直接在 Claude 聊天視窗描述你的錯誤訊息，讓 AI 協助你排解。

進階使用者參考

OpenAI Codex 是什麼？

Codex 是 OpenAI 開發的 AI coding agent，2025 年 4 月以全新形式發布（與 2021 年舊版 Codex API 完全無關）。它以終端機或雲端沙箱為工作環境，能同時並行處理多個任務，特別適合工程化的自動化 pipeline。目前由 GPT-5.3-Codex 模型驅動。

CLI 終端機 agent
命令列操作，適合自動化
多任務並行執行
同時處理多個審圖案件
GitHub / CI/CD 整合
PR 自動提案與 review、CI Autofix
雲端沙箱隔離環境
不需本機安裝，遠端執行
PR 自動提案與 review
自動化程式碼審查流程
CI Autofix 自動修復
偵測問題後自動提交修正

消防圖面 AI 審查流程（兩種工具相同）

DWG / DXF 圖面 ↓ ezdxf / LibreDWG / ODA File Converter 轉換 圖層解析 → 設備辨識 → 座標距離分析 ↓ 法規 Rule Engine 比對 AI 推理 → 產生缺失報告 兩種工具均不能直接讀取 DWG 格式，需透過 parser 轉換成 AI 可理解的格式後再分析

依 Terminal-Bench 2.0 數據：Codex 77.3%、Claude Code 65.4%，故 Codex CLI 略優但兩者均屬強級。Claude 具備原生多模態能力，對 PDF 與截圖的空間理解略優於 Codex。

定位比較

CLAUDE CODE — 超強 AI 工程師

互動式、對話式操作，適合個人日常審圖、一邊問問題一邊分析、需要 AI 解釋推理過程的場景。

CODEX — AI DevOps 團隊

工程化流程、多任務並行，適合夜間批次審圖、建立自動化 pipeline、需要 CI/CD 整合的大型系統。

你的情境 建議工具

需要閱讀 PDF 或截圖理解整體圖面布局 Claude Code
開發 DXF parser / 法規 Rule Engine 程式 Codex 或兩者皆可

建議的自動化審圖系統開發順序（進階）

  1. PDF + OCR PDF 圖說 + 文字辨識 + AI 初步 review，最快可驗證
  2. DXF 幾何解析 ezdxf 解析座標、圖層、設備位置，提供精確數值
  3. 法規 Rule Engine 建立設置標準規則，自動比對並標記違規項目
  4. 自動化 Workflow 批次排程、自動產報告、多案件並行處理
