重要聲明

本手冊中的 AI 工具只能作為輔助初審工具，最終判斷仍需由具備資格與責任的消防設備師或相關專業人員確認。AI 輸出的法規條文、距離計算、設備配置意見，都必須回到原圖、正式法規與專業判斷進行人工複核。本手冊所稱「審圖」是指「AI 輔助初步檢查」，不等同於正式審查、簽證或任何法定審核程序。

第一章：帳號申請與安裝

Step 1：申請 Claude 帳號

Claude 是 Anthropic 公司開發的 AI 助手。使用前需要先在官方網站建立帳號，就像申請電子郵件帳號一樣。

操作步驟：

  • 1. 打開電腦的瀏覽器（Chrome、Edge 或 Firefox 均可）。
  • 2. 在網址列輸入 claude.ai，按 Enter。
  • 3. 頁面會顯示 Claude 官方首頁。點擊畫面中央的「Sign up」按鈕。
  • 4. 在「Email」欄位輸入你的電子郵件地址，然後點擊「Continue」。
  • 5. 系統會發送一封驗證信到你的信箱。打開信箱，找到來自 Anthropic 的信，點擊信中的驗證連結。
  • 6. 回到瀏覽器，依畫面提示設定你的帳號密碼，並填寫基本資料。
  • 7. 完成後頁面會自動進入 Claude 的對話畫面，表示帳號建立成功。

✅ 確認成功的標誌：瀏覽器顯示一個對話框，左下角有一個文字輸入欄位，畫面上方顯示你的帳號名稱或電子郵件。

Step 5：安裝 Claude Code CLI

什麼是 CLI？CLI 是「Command Line Interface」的縮寫，中文叫做「命令列介面」。簡單說，它是一種讓你用文字指令操作電腦的方式，不用滑鼠點擊，直接打字告訴電腦要做什麼。Claude Code 就是透過這種方式運作的——你在一個叫做「終端機」或「PowerShell」的視窗裡打字，Claude Code 就會幫你讀取圖面、分析資料、產生報告。

安裝 Claude Code 之前，需要先安裝 Node.js。Node.js 是一個程式執行環境，可以把它想成是讓 Claude Code 能在你電腦上運作的「底層引擎」。

第一步：安裝 Node.js

  • 1. 打開瀏覽器，前往 nodejs.org。
  • 2. 點擊標示「LTS」的下載按鈕（LTS 代表長期穩定版，適合一般用途）。
  • 3. 下載完成後，執行安裝檔，全程按「Next」與「Install」即可，不需要更改任何設定。
  • 4. 安裝完成後，需要重新啟動電腦一次。

第二步：開啟 PowerShell

PowerShell 是 Windows 內建的終端機程式，就是我們等一下要打字指令的地方。

  • 1. 按下鍵盤左下角的 Windows 鍵（或點擊螢幕左下角的 Windows 圖示）。
  • 2. 在搜尋框輸入「PowerShell」。
  • 3. 在搜尋結果中找到「Windows PowerShell」，對它按滑鼠右鍵。
  • 4. 選擇「以系統管理員身分執行」。
  • 5. 如果出現「是否允許此應用程式對您的裝置進行變更」，點擊「是」。

這時會出現一個深色背景的視窗，裡面有一行文字結尾是 > 符號，表示 PowerShell 已準備好接受你的指令了。

第三章：第一次啟動 Claude Code

什麼是終端機？

終端機（或稱「命令提示字元」、「PowerShell」）是一種純文字的操作視窗。它的外觀就是一個深色背景的視窗，裡面只有文字，沒有按鈕和圖示。你可以把它想像成一個「打字搜尋欄」——不同的是，這個搜尋欄是直接和電腦的作業系統對話，告訴它要做什麼事。Claude Code 就是在這個視窗裡運作的。你打字，Claude Code 就回應。

不需要擔心「打錯指令」會弄壞電腦。Claude Code 的指令都很安全，最壞的情況不過是顯示錯誤訊息，關掉重開就好。

第五章：看懂 AI 的回應

三種判斷狀態的意思

Claude Code 在審查結果中會使用三種標籤，代表不同的判斷程度：

標籤意思你應該怎麼做
✅ 符合AI 根據可見圖面與參考資料，判斷此項目符合要求仍需人工複核，確認 AI 看到的是正確的圖面與正確的標準
❌ 不符合AI 發現明顯的疑點，判斷此項目可能不符合要求優先人工複核，確認 AI 的判斷依據是否正確，再決定是否列為正式缺失
❓ 需確認AI 無法從現有資料判斷，需要人工確認或補充資料列出清單，逐項人工查證，不要忽略這個標籤

當 AI 說不確定時

如果 Claude Code 回覆「我無法從現有資料判斷」或「需要更多資訊」，這是正確行為，表示它沒有憑空猜測。這時候你可以：

  • 提供更清楚的截圖或局部放大圖，再問一次。
  • 在案件說明.md 補充更多背景資訊，然後重新詢問。
  • 接受「需確認」的結論，由人工查證。

當法規條文看起來不對時

AI 有時會引用聽起來很像真實法規、但實際上並不存在或條號有誤的條文，這個現象叫做「幻覺」（hallucination）。遇到這種情況：

  • 不要直接使用 AI 提供的條號。
  • 在你熟悉的法規資料庫或書面資料中查證該條文是否真實存在。
  • 如果查不到，將該審查項目改為「需確認」，並在報告中備註「法規條文待人工查證」。
  • 你可以詢問 Claude Code：「你剛才引用的條文，你能確定這個條號是正確的嗎？若不確定請說明。」這樣的追問通常能讓它更誠實地標示不確定項目。

第六章：人工複核清單

AI 報告產出後，依下列清單逐項人工複核：

  • 確認 AI 讀取的是正確版本的圖面，而非舊版或錯誤檔案。
  • 確認 DXF 單位（公尺或毫米）與案件說明.md 一致。
  • 確認設備圖塊的辨識是否正確（撒水頭是否真的是撒水頭，探測器是否為正確類型）。
  • 確認 PDF 中的文字標註是否被正確讀取，沒有漏字或誤讀。
  • 逐條查證報告中引用的法規條文是否真實存在、條號是否正確。
  • 距離計算結果是否合理，可手算一到兩組數據進行比對。
  • 「不符合」項目是否有明確的圖面依據，不只是 AI 的猜測。
  • 「需確認」項目是否全部列入後續追蹤，沒有遺漏。
  • 報告語氣是否適合內部討論，不包含過於肯定的法律判斷用語。
  • 最終審查結論由具備資格的人員簽名確認，AI 報告僅作為草稿附件。

第七章：常見問題

問題一：輸入 claude 後出現「command not found」錯誤

原因：Claude Code 未正確安裝，或 Node.js 安裝後未重新啟動電腦。

解決方式：

  • 1. 先確認 Node.js 是否安裝成功：在 PowerShell 中輸入 node --version，如果顯示版本號（例如 v20.x.x）表示 Node.js 正常；如果出現錯誤，請重新執行 Step 5 的 Node.js 安裝步驟，安裝後重新啟動電腦。
  • 2. Node.js 確認正常後，重新執行 Claude Code 安裝指令：npm install -g @anthropic-ai/claude-code。
  • 3. 安裝完成後，關閉 PowerShell，重新開啟（以系統管理員身分），再試一次 claude --version。

問題二：Claude Code 說找不到我的檔案

原因：目前的工作路徑不對，或圖面檔案沒有放在正確資料夾。

解決方式：

  • 1. 確認你在 PowerShell 中有執行 cd C:\消防審圖，而且游標前方顯示 C:\消防審圖>。
  • 2. 用 PowerShell 輸入 dir 並按 Enter，確認列出的檔案清單中有你的圖面檔案和 CLAUDE.md。
  • 3. 如果檔案不在清單中，請打開檔案總管，確認圖面已複製到 C:\消防審圖\01_圖面輸入 資料夾內。

問題三：AI 引用的法規條文我查不到

原因：AI 可能產生了不存在的條號（「幻覺」現象）。這在所有 AI 工具中都可能發生，無法完全避免。

解決方式：

  • 查不到的條文，一律改標為「需確認」，不要在正式文件中使用。
  • 詢問 Claude Code：「你剛才引用的條文，你能確定這個條號準確嗎？」讓它自我核對。
  • 在 CLAUDE.md 中已設定「若無法確認條號，請明確說明需要人工查證」，若 AI 沒有遵守，可以再提醒它：「請遵守 CLAUDE.md 中的回答規則，不確定的條文請標示需確認。」

問題四：DXF 距離計算結果看起來不對

原因：最常見的原因是圖面單位未正確設定，導致 AI 以錯誤的單位進行計算。

解決方式：

  • 1. 檢查 CLAUDE.md 與案件說明.md 中的圖面單位是否都填寫正確（公尺或毫米）。
  • 2. 詢問 Claude Code：「你在計算距離時使用的單位是什麼？請說明你的判斷依據。」
  • 3. 手動計算一到兩組距離進行比對，確認倍率是否正確。
  • 4. 如果單位確認有誤，在案件說明.md 修正後，重新要求 Claude Code 進行計算。

問題五：Claude Code 的回應突然中斷

原因：Claude Code（以及所有 AI）有一個「上下文視窗」限制，就是說它能同時記住的資料量有上限。處理很多圖面或非常長的對話時，可能會在回應中途停止。

解決方式：

  • 輸入「請繼續」或「continue」讓它接著說完。
  • 如果它已經忘記前面的內容，可以輸入：「請根據你目前讀取的資料，繼續完成審查報告。」
  • 大型案件建議分段審查：先審一個系統（例如撒水），確認後再審下一個系統（例如探測器），不要一次要求它做全部。

進階工具：Codex

Codex 是 OpenAI 開發的 AI 程式代理工具。和 Claude Code 的主要差異在於，Codex 能自動建立並執行 Python 腳本——也就是說，你給它一個任務，它會自己寫程式、執行程式、讀取輸出，再把結果整理給你。對於需要批次處理多個案件、或希望建立可重複使用的自動化分析流程的使用者，Codex 是很有價值的進階工具。

建議在熟練使用 Claude Code 完成至少十個案件的審圖之後，再開始評估是否需要導入 Codex。Codex 的設定比 Claude Code 複雜，涉及 Python 虛擬環境、套件安裝、腳本除錯等技術細節，對於沒有程式背景的使用者，學習曲線較陡。Codex 使用 AGENTS.md 作為工作設定檔，功能與 Claude Code 的 CLAUDE.md 類似，可以在 OpenAI 官方文件中查閱相關設定方式。

本手冊的範圍僅涵蓋 Claude Code 的完整使用流程。Codex 的詳細說明請參閱 OpenAI 官方開發者文件（前往 platform.openai.com/docs，搜尋「Codex」）。

結語

AI 工具不會取代消防審圖的專業判斷，但它可以幫助你更快整理資料、找出疑點、產生初步意見草稿，讓你把時間留給真正需要人工判斷的地方。本手冊所有步驟都設計成從零開始可以照做，遇到問題歡迎回到第七章查閱常見問題，或直接詢問 Claude Code 本身——「我剛才遇到這個錯誤，請幫我診斷」，往往也能得到有用的提示。祝審圖順利。

⚠️ 提醒：本手冊所有 AI 輸出結果均屬輔助性質。法規條文、距離計算與設備配置判斷，最終皆須由具備資格的消防設備師或相關專業人員進行人工複核與確認，方可作為正式審查意見。
AI 輔助初審結果僅供內部檢查與討論，正式判斷仍需人工複核。本手冊適用對象為消防設備製圖與審圖相關人員。
